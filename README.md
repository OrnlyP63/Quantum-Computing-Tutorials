# Quantum Computing with Qiskit

A complete, multi-track tutorial series covering quantum computing from first principles to hardware-level engineering. Every module pairs a YouTube video with a hands-on Jupyter Notebook generated programmatically using `nbformat`.

---

## What This Is

27 production-ready Jupyter Notebooks across 5 tracks, designed for four distinct audiences. The series starts with shared foundations that everyone completes, then branches into parallel tracks tailored to each learner's background.

The notebooks are not written by hand — each one is generated by a `generate_nb.py` script that uses `nbformat` to construct cells programmatically. This keeps content structured, reproducible, and easy to update.

## Getting Started

### Prerequisites

```
python >= 3.12
```

Install dependencies:

```bash
pip install qiskit qiskit-aer qiskit-ibm-runtime qiskit-machine-learning
pip install numpy matplotlib scipy networkx nbformat
```

### Generate All Notebooks

```bash
find modules -name "generate_nb.py" | sort | while read f; do
  (cd "$(dirname "$f")" && python generate_nb.py)
done
```

Or regenerate a single module:

```bash
cd modules/foundation/F1 && python generate_nb.py
```

### Launch Jupyter

```bash
jupyter lab
# or
jupyter notebook
```

## Notebook Structure

Every notebook follows the same 7-section layout:

1. **Header** — track, module, title, prerequisites, estimated time
2. **Learning Objectives** — 4–5 measurable outcomes
3. **Concept Summary** — theory with LaTeX equations, connection to classical ML/CS
4. **Guided Code Cells** — annotated, runnable, with visualizations
5. **Exercises** — code cells with blanks (`# YOUR CODE HERE`)
6. **Challenge** — one open-ended harder problem, no solution provided
7. **Summary** — key formulas table and what comes next

## Technical Stack

| Component | Choice |
|-----------|--------|
| Quantum SDK | Qiskit 1.x |
| Simulator | `qiskit-aer` (AerSimulator) |
| QML | `qiskit-machine-learning` (SamplerQNN, TorchConnector, FidelityQuantumKernel) |
| Hardware access | `qiskit-ibm-runtime` (FakeManhattan, QiskitRuntimeService) |
| Visualization | matplotlib, matplotlib 3D (Bloch sphere), networkx (coupling maps) |
| Notebook generation | `nbformat` v4 |
| Python | 3.12+ |

All notebooks use the modern Qiskit 1.x API: `from qiskit_aer import AerSimulator`, `transpile()` before `sim.run()`, and `qiskit.quantum_info` primitives throughout.

## Regenerating Notebooks

The `.ipynb` files are outputs — the source of truth is `generate_nb.py`. If you modify a generator script, re-run it from within its directory:

```bash
cd modules/engineer/E5 && python generate_nb.py
```

The script writes the notebook to the same directory.
